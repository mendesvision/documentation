### Parâmetros

<ParamField path="id" type="string" required>
  ID único da transação de pagamento.
</ParamField>

<Note>
  Este endpoint sincroniza automaticamente o status com o provedor de pagamento. Se o status mudar durante a consulta, o campo `statusUpdated` será `true`.
</Note>

### Campos da Resposta

<ResponseField name="qrCode" type="string">
  QR Code em formato base64 (data URI). Pode ser `null` se o QR Code não estiver disponível. Formato: `data:image/png;base64,...`
</ResponseField>

<ResponseField name="transactionId" type="string">
  ID da transação na gateway.
</ResponseField>

### Permissões Necessárias

Esta rota requer a permissão `read:payments`.

### Rate Limiting

- **10 req/s (600 req/min) por API Key**
